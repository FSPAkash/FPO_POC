How farmer signals become
business execution.

01
Input request
Farmer asks for input such as urea
Intent classifier scores item, quantity, and crop context
High-confidence demand becomes a structured input-demand record
Stock check → purchase request or direct issue
Threshold-crossing PRs go to approvals; goods receipt raised on arrival
Full sequence →
02
Harvest readiness
Farmer signals harvest readiness
Crop Cycle Agent creates harvest_signal using keyword strength and timing
High-confidence signals create a produce collection
Market Allocation Agent finds best buyer demand, creates sales order
Within-policy orders dispatch immediately; others hold for approval
Full sequence →
03
Proactive crop-cycle outreach
Orchestration cycle scans active crop seasons
Seasons nearing harvest with no recent collection trigger a proactive message
Each outreach is logged as an agent alert and task
Full sequence →
04
Price enquiry and advisory
Price queries answered from latest crop-mandi snapshot
Missing data triggers a handoff instead of a fabricated reply
Advisory queries use crop reference data and active season timing

Command Centre and Governance

Human control,
always in reach.

The platform acts autonomously within policy — and surfaces anything that needs a human.

Command Centre
The operating surface for agent performance summary, fulfillment queue, harvest watchlist, market queue, human handoffs, recent runs, alerts, and tasks. Everything autonomous — visible at a glance.
Approvals screen
Purchase requests, sales orders, settlements, and any threshold-crossing action route here. Approvers see full context — no chasing systems for background.
Handoff Desk
Concentrates staff attention on judgement work. Low-confidence cases, disputes, disease escalations, and anything crossing an approval threshold land here first.
Policy thresholds
Input autonomy trust, harvest confidence, PR auto-approval, sales-order approval, and outreach window define precisely when autonomy can act and when review is required.
Section 05 · Technical Architecture

A single data spine across
frontend, backend, and audit.

Layer 01
Frontend
React SPA rendering the Command Centre, Farmer Network, Fulfillment, Market Execution, Handoff Desk, Campaigns, Approvals, and Carbon modules.
Layer 02
Backend
Flask API service exposing registry, operations, market, communication, carbon, approvals, and agent-orchestration endpoints over a consistent JSON contract.
Layer 03
Data & Orchestration
A dataset module holds the operational records. A rule-based orchestration layer reads and writes into the same records used by every module, keeping every screen in sync.
Layer 04
Observability
Every orchestration pass is recorded under agent_runs, agent_tasks, agent_alerts, and harvest_signals — every business record carries source_ref and created_by_agent for audit.
